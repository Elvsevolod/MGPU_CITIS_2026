## 3) Интерпретация различий

Isolation Forest оптимизирует обнаружение **общих структурных выбросов**, а не только нарушения правила `< 90 дней`.
Поэтому:

- часть IF-аномалий не является прямым нарушением частоты (но может быть аномалией по другим признакам: школа, вариант, профиль качества, нетипичный паттерн попыток);
- часть rule-based нарушений не попадает в IF-топ из-за более «типичного» контекста остальных признаков.

Практически это означает, что лучше использовать **двухконтурный контроль**:

1. **Hard rules** (обязательные нарушения регламента),
2. **Isolation Forest** (дополнительный риск-контур для ручной проверки).
