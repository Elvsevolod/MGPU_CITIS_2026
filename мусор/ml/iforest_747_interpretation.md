# Интерпретация 747 аномалий Isolation Forest

Источник: `ml/iforest_747_anomalies_with_reasons.csv`, `ml/iforest_747_anomalies_summary.csv`.

## Что модель считает «не таким»

Модель пометила 747 записей как нетипичные не по одному признаку, а по сочетанию признаков профиля записи.

Наиболее частые причины:

- Разные направившая и тестирующая организации — **649 / 747 (86.88%)**
- Нет предыдущего теста (аномалия определяется по другим признакам) — **502 / 747 (67.20%)**
- Редкая направившая организация (очень мало наблюдений в выборке) — **206 / 747 (27.58%)**
- Повторное тестирование раньше 90 дней — **90 / 747 (12.05%)**
- Несколько флагов качества данных (`quality_score >= 2`) — **86 / 747 (11.51%)**
- Критично частое повторение (<30 дней) — **22 / 747 (2.95%)**

## Человеческая трактовка

Проще говоря, IF чаще всего выделяет записи, где:

1. Ребёнка направила одна организация, а тест прошёл в другой, и такой паттерн встречается редко в похожем контексте.
2. Это первая фиксация в истории ребёнка (нет предыдущего теста), но профиль записи при этом нетипичный по школе/варианту/качеству данных.
3. Есть сочетание нетипичных атрибутов (редкая школа, редкий вариант, повышенный quality_score), даже если нет прямого нарушения «<90 дней».
4. В части случаев есть прямое процедурное нарушение частоты, и это тоже попадает в аномалии.

## Важно для интерпретации

- `school_mismatch` (разные организации) не всегда ошибка по бизнес-логике: ребёнок может быть направлен одной школой, а тестироваться на другой площадке.  
- Поэтому аномалии IF — это не «подтверждённые нарушения», а **приоритетный список для проверки**.
- Для контроля лучше использовать 2 контура:
  - жёсткие правила (подтверждённые нарушения регламента),
  - IF как детектор нетипичных кейсов.
