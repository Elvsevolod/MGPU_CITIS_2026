# Validation notes: Isolation Forest tuning

## Что тюнили

Провели сеточный прогон параметров `IsolationForest`:

- `contamination`: `0.02`, `0.03`, `0.05`
- `n_estimators`: `100`, `300`, `500`
- `max_samples`: `1024`, `4096`, `8192`

Сравнение качества делали относительно ручного контура аномалий (`flag_freq_violation_stage3`), то есть:

- `precision_vs_manual` — доля найденных моделью аномалий, которые совпали с ручными;
- `recall_vs_manual` — доля ручных аномалий, покрытая моделью;
- `f1_vs_manual` — баланс precision/recall.

Полная таблица прогона сохранена в `ml/iforest_tuning_results.csv`.

## Что меняется при настройке

- Увеличение `contamination`:
  - повышает число найденных аномалий и `recall`;
  - снижает `precision` (больше ложных срабатываний относительно ручных правил).
- Увеличение `n_estimators`:
  - стабилизирует результат;
  - прирост после `300` деревьев умеренный, но у нас лучший F1 получился на `500`.
- Увеличение `max_samples`:
  - в этом датасете привело к просадке совпадения с ручным контуром;
  - `1024` оказалось заметно лучше `4096/8192` по F1.

## Рекомендации

### 1) Базовая рабочая конфигурация

Для баланса precision/recall:

- `contamination=0.03`
- `n_estimators=500`
- `max_samples=1024`

### 2) Если важнее не пропустить подозрительные кейсы (высокий recall)

- `contamination=0.05`
- `n_estimators=500`
- `max_samples=1024`

Плюс: выше покрытие ручных нарушений. Минус: больше кейсов на ручную проверку.

### 3) Если важнее снизить нагрузку на ручную проверку (выше precision)

- `contamination=0.02`
- `n_estimators=500`
- `max_samples=1024`

Плюс: меньше ложных срабатываний. Минус: ниже покрытие.

## Практический next step

Переключать рабочий профиль в зависимости от операционного режима:

- режим «контроль/аудит» → recall-профиль (`0.05`);
- режим «ежедневный мониторинг» → balanced-профиль (`0.03`);
- режим «ограниченный ресурс проверки» → precision-профиль (`0.02`).

Подход к интерпретации и ключевым параметрам согласован с практикой из обзора по Isolation Forest: [Habr: Выявляем аномалии с помощью Isolation Forest](https://habr.com/ru/companies/otus/articles/881086/).
